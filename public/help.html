How to use Is It Critical?

How to Use This App

  1. Use the search bar to find CVEs by keyword, vendor, or CVE ID
  2. Use the date range filter to narrow results by how recently they were published
  3. Use the severity filter to show only CVEs of a specific severity level
  4. Click any CVE row in the feed to view detailed information about that vulnerability
  5. Click any column header in the CVE feed to sort by that column

Severity Color Key

  • ● Critical: Score 9.0 to 10.0. Extremely dangerous, often remotely exploitable with no authentication required.
  • ● High: Score 7.0 to 8.9. Serious vulnerabilities that are likely to be exploited.
  • ● Medium: Score 4.0 to 6.9. Moderate risk, often requiring some conditions to exploit.
  • ● Low: Score 0.1 to 3.9. Minimal risk, typically difficult to exploit or limited in impact.

FAQ

Why are there so many vulnerabilities?

Software vulnerabilities are discovered constantly across thousands of products worldwide. The NVD receives hundreds of new CVE submissions every day from researchers, vendors, and automated scanning tools.

How often is the data updated?

This app syncs with the NVD using the "Sync" button on the home page to keep data fresh. The database reflects CVEs published within the last 120 days.

Why do some CVEs show N/A for severity?

Some CVEs are still awaiting analysis by NVD staff and have not yet been assigned a CVSS score. Others may have been published before the CVSS scoring system existed.

Why does my search return no results?

Try broadening your search. Use a shorter keyword, change the date range to a longer window, or set severity to All Severities. The database contains CVEs from the last 120 days so very old CVEs will not appear.
